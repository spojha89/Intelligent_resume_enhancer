Resume Generator-

Tools-
Lambda, OpenSearch, S3, Bedrock models, FastAPI

Implementation:

1. Ingestion of current resume and job description to OpenSearch
2. Use bedrock models to generate resume 
3. Extract to S3

Component usage:
-> Lambda to ask to Upload resume and job description to S3
→ Lambda (Extract Requirements + Generate Query Embedding+ Load to OpenSearch)
-> Lambda extract current resume contents + Generate Query Embedding+ Load to OpenSearch
-> Lambda to use bedrock model to find matching probability of resume as per job description
→ Lambda (Call Bedrock with Context) and call OpenSearch (Retrieve Relevant Experiences)
→ Lambda to generate customized resume using Bedrock Claude
→ Lambda to generate report in format as PDF/DOCX to S3
-> Step function to orchestrate between lambda functions 

Architecture-
User
 └── Upload Resume + JD
       ↓
S3 (Raw Files) --> done 
       ↓
Step Functions Orchestrator
 ├── Lambda 1: Parse Resume (sections, bullets)
 │      └── Generate embeddings → OpenSearch
 │
 ├── Lambda 2: Parse JD + Extract Requirements
 │      └── Generate JD query embedding (NO storage)
 │
 ├── Lambda 3: Matching & Scoring
 │      ├── Vector similarity
 │      ├── Skill overlap
 │      └── Produce match score
 │
 ├── Lambda 4: Context Retrieval
 │      └── OpenSearch kNN using JD embedding
 │
 ├── Lambda 5: Resume Rewrite (Claude)
 │      └── Section-wise generation
 │
 ├── Lambda 6: Verification / Guardrails (optional)
 │
 ├── Lambda 7: PDF/DOCX Generator
 │      └── Upload final resume to S3
 │
 └── Persist metadata (DynamoDB)
